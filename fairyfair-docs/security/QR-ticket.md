## Типы QR-кодов

Каждый QR-код содержит **уникальный зашифрованный токен**. Тип токена и набор полей внутри него зависят от роли участника.

---

### Билет посетителя

Выдаётся при покупке билета на `fairyfair.ru`.

| Поле | Описание |
|------|----------|
| `login` | Логин пользователя (или `anonymous` для покупки без аккаунта) |
| `full_name` | ФИО покупателя |
| `ticket_number` | Порядковый номер билета в заказе |

> **Важно:** Один заказ может содержать несколько билетов. Каждый билет получает отдельный QR-код с уникальным порядковым номером.

Билет является **пропуском входа/выхода** — повторные сканирования допустимы и логируются как отдельные записи.

---

### Аккредитация торговца

Выдаётся после одобрения заявки на `staff.fairyfair.ru` и оплаты торгового места.

| Поле | Описание |
|------|----------|
| `shop_name` | Название торговой точки |
| `table_number` | Номер арендованного стола |
| `full_name` | ФИО участника команды |

> Если у торговой точки несколько представителей — каждый получает **свой отдельный QR-код** с личными данными и одним номером стола.

---

### Аккредитация косплеера

Выдаётся после одобрения заявки на `staff.fairyfair.ru`.

| Поле | Описание |
|------|----------|
| `full_name` | ФИО косплеера |
| `character` | Название персонажа |

---

### Аккредитация фотографа

Выдаётся после одобрения заявки на `staff.fairyfair.ru`.

| Поле | Описание |
|------|----------|
| `full_name` | ФИО фотографа |
| `role` | Фиксированное значение: `"фотограф"` |

---

### Аккредитация музыканта

Выдаётся после одобрения заявки на `staff.fairyfair.ru`.

| Поле | Описание |
|------|----------|
| `band_name` | Название группы или исполнителя |
| `full_name` | ФИО конкретного участника |

> Каждый участник группы получает **отдельный QR-код** с общим названием коллектива и личными данными.

## Алгоритм генерации

```
Событие: покупка билета / одобрение аккредитации
          │
          ▼
1. Формирование payload
   Сбор данных согласно типу билета (см. таблицы выше)
          │
          ▼
2. Шифрование токена
   Payload шифруется → генерируется уникальный токен
          │
          ▼
3. Запись в базу данных
   Сохраняется: токен, тип, данные участника,
   срок действия (следующий день после окончания фестиваля)
          │
          ▼
4. Генерация QR-кода
   QR-код создаётся на основе зашифрованного токена
          │
          ▼
5. Доставка
   QR-код отправляется пользователю на email
   (для участников staff — также доступен в личном кабинете)
```

## Жизненный цикл билета

```
[Создан] → [Активен] → [Истёк]
               │
               ├── Использован (сканирование охраной)
               │
               └── Отозван (вручную организатором / возврат)
```

| Статус | Описание |
|--------|----------|
| **Активен** | Билет действителен, проход разрешён |
| **Истёк** | Дата окончания фестиваля прошла — QR более недействителен |
| **Отозван** | Организатор аннулировал билет (например, при возврате) |

> Срок действия билета устанавливается автоматически: **следующий день после даты окончания фестиваля**. Настраивается в модуле «Билеты» admin-панели.

## Верификация на входе

Сканирование QR-кодов выполняется охраной через [Android-приложение](./fairyfair-docs/domains/android.md).

**Онлайн-режим:**

```
Сканирование QR → расшифровка токена → проверка по глобальной БД
                                                    │
                        ┌───────────────────────────┤
                        ▼                           ▼
                  Токен валиден               Токен недействителен
                  Зелёный экран               Красный экран + причина
                        │
                        ▼
               Лог записан в глобальную БД
```

**Офлайн-режим** (при потере сети):

```
Сканирование QR → проверка по локальной копии БД
                        │
                        ▼
               Результат на экране
                        │
                        ▼
               Запись в локальную очередь
                        │
                 (при восстановлении сети)
                        ▼
               Синхронизация с глобальной БД
```

**Результаты сканирования:**

| Экран | Причина |
|-------|---------|
| Зелёный | Билет / аккредитация действительны |
| Красный | Билет не найден, аннулирован или истёк |
| Красный | Аккредитация ожидает одобрения |
| Красный | Аккредитация отозвана |
