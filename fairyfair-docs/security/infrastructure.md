# Инфраструктурная изоляция

## Физическое разделение admin-панели

`admin.fairyfair.ru` закрыт от публичной сети. Доступ возможен только через:

- **VPN** (корпоративная сеть команды), или
- **Именной сертификат** (выдаётся IT-администратором индивидуально)

Это означает: даже при обнаружении уязвимости в публичной части (`fairyfair.ru` или `staff.fairyfair.ru`) вектор атаки на admin-панель **не открывается автоматически** — злоумышленник сначала должен преодолеть сетевую изоляцию.

```
Интернет
    │
    ├── fairyfair.ru       (публичный)
    ├── staff.fairyfair.ru (публичный)
    │
    └── admin.fairyfair.ru (только VPN / сертификат)
              │
              └── + обязательная 2FA
```

## SSH-доступ к серверу

- Аутентификация **только по SSH-ключам** — парольная аутентификация отключена
- **fail2ban** автоматически блокирует IP-адреса при превышении порога неудачных попыток входа

---

## Безопасность платёжных вебхуков

Каждый входящий вебхук от платёжного шлюза проходит **три уровня проверки**:

```
Входящий запрос от шлюза
        │
        ▼
1. IP-whitelist
   Принимается только с подтверждённых IP-адресов провайдера
        │
        ▼
2. HMAC-подпись
   Проверка целостности и подлинности payload
        │
        ▼
3. Pydantic-валидация
   Структурная проверка тела запроса
        │
        ▼
   Обработка платежа
```

Любой запрос, не прошедший хотя бы одну проверку, отклоняется с логированием инцидента.
