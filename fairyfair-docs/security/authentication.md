# Аутентификация и управление сессиями

## Хеширование паролей — Argon2

Используется **Argon2** — победитель Password Hashing Competition 2015.

- Memory-hard алгоритм: делает GPU/ASIC-атаки экономически нецелесообразными
- Применяется ко всем пользовательским паролям на всех доменах платформы

---

## JWT-токены

Стандарт: **HS256**

Обязательные поля каждого токена:

| Поле | Назначение |
|------|-----------|
| `jti` | Уникальный идентификатор для инвалидации конкретного токена |
| `type` | Разделение access / refresh токенов |
| `exp` | Время истечения |

Дополнительная защита: каждый токен привязан к **IP-адресу**, **User-Agent** и **fingerprint устройства** — это позволяет детектировать использование токена с нетипичного устройства или сети.

---

## Refresh-токены

- Хранятся в БД в виде **SHA-256 хеша**, не оригинала
- **Ротация** при каждом обновлении — старый токен инвалидируется
- Поддержка явного отзыва (revoke):
  - при смене пароля
  - при выходе со всех устройств
  - при деактивации аккаунта администратором

## Cookie

| Флаг | Значение | Назначение |
|------|---------|-----------|
| `HttpOnly` | `true` | Cookie недоступны через JavaScript — защита от XSS |
| `Secure` | `true` (в production) | Передача только по HTTPS |
| `SameSite` | `Lax` | Защита от CSRF при кросс-сайтовых запросах |
