# Мониторинг и обнаружение атак

## Аудит-лог

Каждое действие в системе логируется автоматически:

| Поле записи | Описание |
|-------------|----------|
| Пользователь | Кто выполнил действие |
| Действие | Что именно было сделано |
| Временная метка | Когда произошло |
| IP-адрес | С какого адреса |

Лента аудита доступна в **дашборде** `admin.fairyfair.ru`. IT-администратор может фильтровать лог по пользователю, типу события и временному диапазону.

---

## Telegram-оповещения

Критические события автоматически эскалируются в Telegram ответственным сотрудникам:

| Событие | Получатель |
|---------|-----------|
| Подозрительные попытки входа | IT-администратор |
| Сработавшие ловушки (honeypot) | IT-администратор |
| Аномалии в платёжных операциях | Бухгалтер / организатор |
| Новый тикет поддержки (ночное время) | Дежурный волонтёр |
| Ошибки синхронизации (Android-приложение) | IT-администратор |

---

## Реакция на инциденты

При выявлении аномалии в аудит-логе IT-администратор может:

1. Деактивировать аккаунт подозрительного пользователя немедленно
2. Отозвать VPN-сертификат или токен
3. Принудительно завершить все активные сессии
4. Уведомить организатора о произошедшем

Деактивация аккаунта: CRM → «Пользователи» → карточка сотрудника → «Деактивировать»
