# Сторонние интеграции

## VK API

Используется для работы с аудиторией ВКонтакте:

- Публикация анонсов и новостей мероприятия
- Кросспостинг из ленты fairyfair.ru

Настройка: `admin.fairyfair.ru` → Настройки → Ключи и API → VK API

---

## VK OAuth2

Авторизация пользователей через аккаунт ВКонтакте на `fairyfair.ru` и `staff.fairyfair.ru`.

- Упрощает регистрацию — пользователь не вводит email и пароль вручную
- Данные профиля подтягиваются автоматически

Настройка: `admin.fairyfair.ru` → Настройки → Ключи и API → VK OAuth2

---

## Telegram API

Используется для уведомлений команды организаторов:

| Событие | Получатель |
|---------|-----------|
| Новый тикет поддержки | Дежурный оператор / волонтёр |
| Критическое событие безопасности | IT-администратор |
| Аномалия оплаты | Бухгалтер / организатор |
| Ночной тикет без ответа | Дежурный |

Настройка: `admin.fairyfair.ru` → Настройки → Ключи и API → Telegram API

## Платёжные шлюзы

Поддерживается подключение одного или нескольких шлюзов:

| Шлюз | Статус |
|------|--------|
| ЮКасса (YooKassa) | Поддерживается |
| Тинькофф | Поддерживается |
| Другие шлюзы | Настраивается через API |

### Безопасность платёжных вебхуков

Каждый входящий вебхук от платёжного шлюза проходит тройную проверку:

1. **IP-whitelist** — запрос принимается только с подтверждённых адресов провайдера
2. **HMAC-подпись** — проверка целостности и подлинности payload
3. **Pydantic-валидация** — структурная проверка тела запроса

Настройка: `admin.fairyfair.ru` → Настройки → Ключи и API → Платёжный шлюз

---

## Добавление новой интеграции

Все API-ключи и токены хранятся в модуле **Настройки → Ключи и API**. При обновлении ключа новый ключ применяется только после успешного теста соединения — старый остаётся активным до подтверждения.
