# Android-приложение — Охрана

Отдельное Android-приложение для верификации посетителей и участников на входе. Доступ предоставляется IT-администратором через токен.

## Назначение

Сотрудники охраны используют приложение для проверки QR-кодов билетов и аккредитаций участников. Приложение не имеет доступа к другим модулям системы.

---

## Принцип работы

1. Сотрудник наводит камеру на QR-код посетителя или участника
2. Приложение расшифровывает данные QR-кода
3. Происходит сверка с базой данных билетов
4. На экране отображается результат: **зелёный** (пропустить) или **красный** (отклонить) с указанием причины
5. Факт сканирования записывается в лог с меткой времени

> Билет является **пропуском** — посетитель может входить и выходить. Повторные сканирования одного билета допустимы и логируются как отдельные записи.

---

## Офлайн-режим

При нестабильном или отсутствующем соединении приложение автоматически переключается в офлайн-режим.

**Механизм:**
- При подключении к сети — скачивается локальная копия базы данных билетов
- В офлайне — проверка происходит по локальной копии
- Все события сканирования записываются в локальную очередь
- При восстановлении сети — очередь автоматически синхронизируется с глобальной БД

```
Онлайн:   QR → глобальная БД → результат → лог в глобальной БД
Офлайн:   QR → локальная БД → результат → локальная очередь → синхронизация при восстановлении сети
```

---

## Верификация участников staff

Помимо билетов посетителей, приложение проверяет QR-коды аккредитации участников (торговцы, музыканты, косплееры, фотографы).

При сканировании аккредитации отображается:
- Имя участника
- Роль (торговец / музыкант / косплеер / фотограф)
- Номер торгового места или слота выступления

Статусы аккредитации:
- **Одобрена** — зелёный экран, пропустить
- **Ожидает одобрения** — красный экран, проход запрещён
- **Отозвана** — красный экран с причиной

---

## Авторизация

- Приложение скачивается по ссылке, выданной IT-администратором
- Вход осуществляется по токену доступа
- Токен выдаётся и отзывается через CRM → «Пользователи» в admin-панели
